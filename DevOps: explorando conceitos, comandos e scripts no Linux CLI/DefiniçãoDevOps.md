DevOps (a junção de Development/Desenvolvimento e Opserations/Operações) é uma cultura, uma filosofia e um conjunto de práticas que une as equipes de criação de software e de infraestrutura de TI. [1, 2, 3] 
O objetivo principal é quebrar as barreiras (os chamados "silos") entre quem cria o sistema e quem o mantém funcionando, permitindo entregar aplicações com mais rapidez, segurança e qualidade. [3, 4, 5] 
Aqui estão os pilares que formam o DevOps:
## 1. Cultura e Colaboração
No modelo tradicional, desenvolvedores queriam lançar funcionalidades novas rapidamente, enquanto a equipe de operações focava na estabilidade, gerando conflitos. O DevOps promove uma mudança de mentalidade, onde todos compartilham a mesma responsabilidade: entregar valor ao usuário final de ponta a ponta. [2, 4] 
## 2. Automação
Em vez de fazer processos manuais e repetitivos (que são lentos e propensos a erros), o DevOps utiliza ferramentas para automatizar desde os testes do código até a sua publicação no servidor. [6, 7] 

* CI (Integração Contínua): O código criado pelos desenvolvedores é testado e juntado automaticamente.
* CD (Entrega/Implantação Contínua): O código aprovado vai para produção de forma segura e quase instantânea. [6, 8, 9, 10, 11] 

## 3. Monitoramento e Feedback
O trabalho não termina quando o software é lançado. As equipes monitoram continuamente o sistema para identificar e corrigir falhas antes que o cliente perceba, gerando um ciclo constante de melhoria. [6, 7, 12, 13] 
## Por que as empresas usam DevOps?

* Velocidade: Permite lançar novas funcionalidades e corrigir bugs rapidamente.
* Confiabilidade: A automação e os testes garantem que as atualizações não quebrem o sistema.
* Colaboração: Melhora a harmonia e a comunicação interna entre diferentes times técnicos. [1, 4, 6, 14, 15]
